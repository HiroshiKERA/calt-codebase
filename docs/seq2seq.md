## Seq2Seq training with your own dataset (without calt internal representation)

This guide explains how to train a generic sequence-to-sequence model using datasets generated by external tools (e.g., SageMath), without converting them into the calt internal representation. You will use calt's `StandardDataset` together with a simple pass-through preprocessor and Hugging Face's `Trainer`.

### What you'll build
- Load your data with `StandardDataset`
- Use an identity (no-op) preprocessor so that input text equals output text
- Tokenize with a standard Hugging Face tokenizer
- Train a seq2seq model (e.g., T5) via `Trainer`

---

## Dataset format
`StandardDataset.load_file(path, preprocessor)` expects a plain text file where each line contains one training example in the following format:

```text
<input_text>#<target_text>
```

For example:
```text
2*x0^2 + 3*x1#5*x0^2 + 3*x1
a|b|c#c|b|a
```

Notes:
- Lines without the `#` delimiter are skipped.
- Empty lines are skipped.
- You can pass `max_samples=-1` to load all samples; any non-positive value except `-1` is invalid.

---

## Loading with StandardDataset and a custom Preprocessor
`StandardDataset` delegates transformation to a `Preprocessor` implementing `AbstractPreprocessor`.

- The simplest option is an identity preprocessor that returns the input as-is for both encoding and decoding:

```python
from calt.data_loader.utils.preprocessor import AbstractPreprocessor

class IdentityPreprocessor(AbstractPreprocessor):
    def __init__(self) -> None:
        super().__init__(num_variables=0, max_degree=0, max_coeff=1)

    def encode(self, text: str) -> str:
        return text

    def decode(self, tokens: str) -> str:
        return tokens
```

- To load your dataset files:

```python
from calt.data_loader.utils.data_collator import StandardDataset

preprocessor = IdentityPreprocessor()
train_ds = StandardDataset.load_file("path/to/train.txt", preprocessor=preprocessor)
eval_ds = StandardDataset.load_file("path/to/eval.txt", preprocessor=preprocessor)
```

### What StandardDataCollator does
`StandardDataCollator(tokenizer)` tokenizes `{"input", "target"}` fields into:
- `input_ids`, `attention_mask`
- `decoder_input_ids`, `decoder_attention_mask`
- `labels` (shifted, with padding tokens set to `-100`)

This layout matches seq2seq losses used by models like T5 and BART.

---

## Using a Hugging Face tokenizer and model (T5 example)
```python
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM

tokenizer = AutoTokenizer.from_pretrained("t5-small", use_fast=True)
model = AutoModelForSeq2SeqLM.from_pretrained("t5-small")
```

Any compatible seq2seq model can be used (e.g., `t5-small`, `t5-base`, `facebook/bart-base`).

## End-to-end training via Trainer
There is a draft reference script that wires these pieces together: `scripts/train/seq2seq_train.py`.

### Run from the command line (draft; subject to change)
```bash
python scripts/train/seq2seq_train.py \
  --model_name_or_path t5-small \
  --train_file dataset/sagemath/polynomial_problem/GF7_n=2/train_raw.txt \
  --eval_file dataset/sagemath/polynomial_problem/GF7_n=2/train_raw.txt \
  --per_device_train_batch_size 8 \
  --per_device_eval_batch_size 8 \
  --num_train_epochs 1 \
  --learning_rate 5e-5 \
  --output_dir outputs/seq2seq-training
```

Key arguments (subject to change):
- `--model_name_or_path`: Any Hugging Face seq2seq checkpoint (e.g., `t5-small`)
- `--tokenizer_name_or_path` (optional): If omitted, defaults to `model_name_or_path`
- `--train_file`, `--eval_file`: Paths to text files in the `<input>#<target>` format
- `--per_device_{train,eval}_batch_size`, `--num_train_epochs`, `--learning_rate`: Standard training hyperparameters
- `--output_dir`: Where checkpoints and tokenizer are saved

### What the draft script does internally
- Instantiates `IdentityPreprocessor` and loads datasets with `StandardDataset.load_file`
- Builds a tokenizer and a `AutoModelForSeq2SeqLM`
- Uses `StandardDataCollator(tokenizer)` for dynamic padding and label shifting
- Configures `TrainingArguments` (e.g., `eval_strategy`, `save_strategy`)
- Trains via `Trainer` and optionally evaluates
- Saves model and tokenizer to `output_dir`

---

## Tips
- Start with a small model (e.g., `t5-small`) and batch size to validate the pipeline.
- Ensure your data strictly follows the `<input>#<target>` format; malformed lines are skipped.
- If you need metrics (e.g., token-level accuracy), add a `compute_metrics` function to the `Trainer`.
- For long sequences, adjust `--max_length` at tokenizer call sites if you extend the script.
